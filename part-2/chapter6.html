Chapter 6 – Markov Decision Processes

The Markov decision process (MDP) is the canonical model for sequential decision making under uncertainty. In reinforcement learning it plays the same foundational role that probability distributions play in statistics: it specifies the interaction between an agent and its environment over time. The agent moves through a set of states, selects actions, receives rewards and transitions to new states according to a stochastic transition model. In this chapter we formalise MDPs, introduce the value functions and Bellman operators that underlie dynamic programming, and show how to compute optimal policies via policy and value iteration. Throughout we use small pricing and decision‑making examples rather than robotics to illustrate the ideas.

6.1 MDP formalism

An MDP is defined by a finite or countable set of states S, a set of actions A, a transition probability distribution Pa(s,s′) describing the probability of moving from state s to state s′ when action a is executed, and an immediate reward function Ra(s,s′) giving the payoff for that transition. In the canonical discounted formulation the quadruple (S, A, Pa, Ra) defines a stochastic control process with Markov dynamics: the next state depends only on the current state and action【567934665676015†L192-L218】. The agent also chooses a policy π, a mapping from states to actions that can be deterministic or stochastic, and we often write at = π(st).

The objective in an MDP is to maximise the expected cumulative reward over time. When rewards may extend into the future one discounts later rewards by a factor γ ∈ [0,1]. The expected discounted return for a policy π starting in state s is \[V^π(s) = E[∑t=0^∞ γ^t Ra_t(s_t,s_{t+1}) | s₀ = s, a_t = π(s_t)],\] where the expectation is taken over trajectories generated by the transition dynamics. Choosing a good policy therefore involves balancing immediate rewards against future gains. A policy that maximises the above objective for all states is called an optimal policy【567934665676015†L236-L272】.

An equivalent but sometimes more intuitive formulation uses a five‑tuple (S, A, T, R, γ) where T(s,a,s′) denotes the probability of transitioning to s′ from s under action a and R(s,a,s′) is the immediate reward. In practice one often works with a simulator or generative model that samples (s′,r) ∼ G(s,a) rather than storing the full transition matrix【567934665676015†L275-L297】. In all cases the Markov property implies that the probability of the next state depends only on the current state and action, not on previous history.

To give a concrete example imagine a retailer choosing the price of a product each day. The state s might encode the current day and remaining inventory, the action a is the price to charge, the transition probabilities capture stochastic demand causing inventory to decrease, and the reward is the profit obtained. The retailer’s goal is to choose prices to maximise the discounted sum of expected profits. We will revisit pricing examples throughout this chapter.

6.2 Value functions and Bellman operators

To reason about an MDP one defines value functions that quantify the long‑term quality of states and actions. The state‑value function V^π(s) for a policy π is the expected discounted return starting from state s and following π. Similarly the action‑value function Q^π(s,a) is the expected return starting from state s, taking action a initially and following π thereafter: \[Q^π(s,a) = E[ Ra(s,s₁) + γ V^π(s₁) ],\] where s₁ is drawn from the transition distribution for action a. These functions satisfy a family of recursive relationships known as the Bellman expectation equations. For the state‑value function the equation reads \[V^π(s) = R(s,π(s)) + γ ∑s′ P_{π(s)}(s,s′) V^π(s′),\] and for the action‑value function \[Q^π(s,a) = R(s,a) + γ ∑s′ P_a(s,s′) Q^π(s′,π(s′)).\]

These equations imply that the value functions are fixed points of a mapping called the Bellman operator. Denote by 𝔅^π the operator that maps a function Q to 𝔅^π[Q] defined by the right‑hand side of the Bellman equation. One can show that 𝔅^π is a contraction mapping: for any two functions Q and U we have ‖𝔅^π[Q] − 𝔅^π[U]‖∞ ≤ γ ‖Q − U‖∞. Consequently repeated application of the Bellman operator starting from an arbitrary function converges to the unique fixed point Q^π【221921659924593†L21-L63】.

The contraction property not only guarantees convergence but also provides a way to quantify the rate of convergence: the error decreases geometrically by a factor of γ at each iteration. In practice the Bellman operator is the workhorse behind dynamic programming algorithms such as policy evaluation, policy iteration and value iteration. Understanding the value functions and the Bellman operator is therefore fundamental before we can discuss algorithms for computing optimal policies.

6.3 Policy evaluation

Policy evaluation is the task of computing the value function V^π for a given policy π. Because the value function appears on both sides of the Bellman expectation equation, one typically solves it via iterative methods. Starting with an initial estimate V₀(s) = 0 for all states, one repeatedly applies the Bellman expectation update \[V_{k+1}(s) ← R(s,π(s)) + γ ∑s′ P_{π(s)}(s,s′) V_k(s′)\] until the changes between successive iterates become negligible. This procedure is guaranteed to converge to the true value function because the underlying Bellman operator is a contraction【221921659924593†L21-L63】. The same technique can be applied to action values Q^π.

The following Python function implements policy evaluation for a small pricing MDP. The state space consists of inventory levels 0,1,…,m; the action space contains two prices: a high price and a low price. Demand at the low price is higher on average but yields less profit per unit. We assume the policy always chooses the high price when inventory is low and the low price when inventory is high. The function returns the estimated value of each inventory level.

import numpy as np

# Transition and reward models for a simple pricing MDP
# states: inventory levels 0..m, actions: 0=low price, 1=high price
m = 3
P = np.zeros((2, m+1, m+1))
R = np.zeros((2, m+1, m+1))
# populate transition and reward matrices
for s in range(m+1):
    for a in [0, 1]:
        # demand probabilities: low price sells 0,1,2 units; high price sells 0 or 1 unit
        if a == 0:
            probs = [0.2, 0.5, 0.3]  # low price demand distribution
            reward_per_unit = 5
        else:
            probs = [0.5, 0.5]      # high price demand distribution
            reward_per_unit = 8
        for units_sold, prob in enumerate(probs):
            next_s = max(0, s - units_sold)
            P[a, s, next_s] += prob
            R[a, s, next_s] += prob * units_sold * reward_per_unit

# fixed policy: if inventory ≥ 2 choose low price (0), else choose high price (1)
policy = np.array([1 if s < 2 else 0 for s in range(m+1)])

def policy_evaluation(policy, P, R, gamma=0.9, tol=1e-4):
    V = np.zeros(P.shape[1])
    while True:
        delta = 0
        for s in range(P.shape[1]):
            a = policy[s]
            v_new = (R[a, s] + gamma * P[a, s] @ V).sum()
            delta = max(delta, abs(v_new - V[s]))
            V[s] = v_new
        if delta < tol:
            break
    return V

values = policy_evaluation(policy, P, R)
print("Inventory values under the fixed policy:", values)

Running this code prints the long‑term value of each inventory level when following the specified pricing policy. More sophisticated environments with richer state and action spaces are solved in the same way; the key is to compute expected rewards and discounted values under the transition dynamics and policy.

6.4 Policy and value iteration

Computing an optimal policy requires more than evaluating a fixed policy; one must improve the policy based on the value estimates. The two classical dynamic programming algorithms are value iteration and policy iteration. Value iteration directly seeks the optimal value function V* by repeatedly applying the Bellman optimality update \[V_{k+1}(s) ← maxa [ R(s,a) + γ ∑s′ P_a(s,s′) V_k(s′) ],\] which follows from the Bellman optimality equation【862037459657812†L87-L110】. Once the value function converges, the optimal policy is obtained by choosing for each state the action that attains the maximum on the right‑hand side. Value iteration is simple to implement but performs a full maximisation over actions at every state at each iteration.

Policy iteration, by contrast, alternates two phases: policy evaluation (as described above) and policy improvement. For a current policy π one first computes its value function V^π. Then one improves the policy by greedily selecting at each state the action that maximises the expected return according to V^π: \[π′(s) = argmaxa [ R(s,a) + γ ∑s′ P_a(s,s′) V^π(s′) ].\] Policy iteration repeats evaluation and improvement until the policy no longer changes; it can converge more quickly than value iteration because it avoids repeatedly re‑maximising the value function【862037459657812†L112-L133】. In practice one may use a hybrid method, modified policy iteration, that performs a few evaluation steps between improvements.

The following code implements value iteration for the pricing MDP defined earlier. It computes the optimal price to charge at each inventory level. After convergence, the resulting policy recommends the high or low price depending on whether the marginal value of inventory outweighs the immediate reward.

def value_iteration(P, R, gamma=0.9, tol=1e-4):
    num_states = P.shape[1]
    num_actions = P.shape[0]
    V = np.zeros(num_states)
    while True:
        delta = 0
        for s in range(num_states):
            v_old = V[s]
            # compute value for all actions
            q_sa = np.zeros(num_actions)
            for a in range(num_actions):
                q_sa[a] = (R[a, s] + gamma * P[a, s] @ V).sum()
            V[s] = np.max(q_sa)
            delta = max(delta, abs(v_old - V[s]))
        if delta < tol:
            break
    # derive policy
    policy = np.zeros(num_states, dtype=int)
    for s in range(num_states):
        q_sa = [(R[a, s] + gamma * P[a, s] @ V).sum() for a in range(num_actions)]
        policy[s] = int(np.argmax(q_sa))
    return V, policy

V_opt, policy_opt = value_iteration(P, R)
print("Optimal inventory values:", V_opt)
print("Optimal pricing policy (0=low, 1=high):", policy_opt)

When applied to our small inventory example the optimal policy typically prices high when the inventory is low to maximise immediate profit and prices low when inventory is plentiful to encourage sales. The dynamic programming algorithms described above thus allow us to compute optimal policies in tabular settings. In larger state spaces one turns to approximate dynamic programming and reinforcement learning algorithms discussed in subsequent chapters.
